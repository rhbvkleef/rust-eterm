pub static STRING_EXT: u8

String does not have a corresponding Erlang representation, but is an optimization for sending lists of bytes (integer in the range 0-255) more efficiently over the distribution.

As field Length is an unsigned 2 byte integer (big-endian), implementations must ensure that lists longer than 65535 elements are encoded as LIST_EXT.

Binary representation

1 byte2 bytesLength bytes
107LengthCharacters

The characters are UTF-8 encoded characters. There are a maximum of 65535 bytes, however many characters that turns out to be. It is not a maximum of 65535 characters!
